# はじめての Eidra

Eidra は、AIツールのための安全フィルターです。

あなたの AI ツールとインターネットの間に入って、「いま外へ出ようとしているもの」をチェックし、危ないものを隠したり止めたりローカルに逃がしたりします。

## ひとことで言うと

> Eidra は、AI の見張り番です。

AI が送ろうとしている内容や実行しようとしていることを見て、危ない部分だけ先に対応します。

## なぜ必要なのか

いまの AI ツールは便利です。
でも便利なのは、たくさん見えるからです。

つまり逆に言うと、見えすぎることがあります。

たとえば:

- `.env` に入っている API キー
- プロンプトに混ざった社内コードや顧客情報
- MCP 経由で触れるファイル、DB、shell

こういうものが、本人が気づかないまま外へ出ることがあります。
Eidra はそれを見えるようにします。

## Eidra がやること

### 1. 何が起きているか見せる

AI の通信や検出結果、判断をターミナルのダッシュボードで見られます。

### 2. 秘密情報を隠す

API キー、token、password、個人情報などを見つけたら、外へ出る前にマスクできます。

### 3. 危ないものを止める

危険なリクエストや MCP の tool call は、実行前に block できます。

### 4. 機密度の高いものをローカルへ逃がす

クラウドに送りたくないリクエストは、互換な local model へ route できます。

## たとえるなら

AI は優秀だけど、ちょっと口が軽い部下みたいなものです。

Eidra はドアの前に立っている人です。

- 「これは送って大丈夫です」
- 「この API キーは隠してから送ります」
- 「そのコマンドは危ないので止めます」
- 「これはローカルだけで処理しましょう」

## どんな人に向いているか

- AIコーディングツールをこれから使う人
- Cursor、Claude Code、Codex、各種 SDK を使う開発者
- MCP を agent に繋いでいる人
- AI ワークフローの信頼境界を自分の手元に戻したい人

## いちばん短い試し方

```bash
curl -sf eidra.dev/install | sh
eidra init
eidra doctor
eidra setup codex
eidra dashboard
```

Cursor や Claude Code を使うなら、`codex` を `cursor` や `claude-code` に置き換えるだけです。
