## Core Message

Eidra is the local-first trust layer for AI development.

It helps people:

- see what leaves their machine
- decide what is allowed to leave
- prove what happened later

## Message Hierarchy

### Top-level category

Local-first trust layer for AI development

### Product statement

Eidra is an open-source local proxy and MCP firewall that scans, masks, blocks, routes, and explains sensitive AI traffic before it leaves your machine.

### Functional outcome

Developers can keep using their existing AI tools while gaining a local trust boundary in front of prompts, requests, and tool calls.

## 30-Second Version

Eidra is an open-source local-first trust layer for AI development. It sits in front of tools like Cursor, Claude Code, Codex, SDK workflows, and MCP-connected tools, scans what is about to leave your machine, applies local policy, and can mask, block, or route sensitive traffic before it goes out.

## 2-Minute Version

Modern AI development is no longer just a prompt in a chat box. It now includes editors, coding agents, SDK apps, local models, CI pipelines, and MCP-connected tools. That means developers are sending more context and granting more tool access than ever, often without seeing exactly what is leaving the machine. Eidra is a local-first trust layer built for that shift. It provides a local proxy, a policy engine, a live dashboard, audit logging, and an MCP firewall so developers can inspect, control, and explain what their AI workflows are doing.

## Objection Handling

### "Does this replace my AI tool?"

No. Eidra sits in front of the tools you already use.

### "Do I need another cloud service for this?"

No. Eidra is designed to be local-first.

### "Is this only for security teams?"

No. It is useful for solo developers, builders working with MCP, and teams that need a better trust boundary.

### "Will this be painful to try?"

It is designed to support gradual adoption: scan, inspect, then add stronger enforcement when you want it.
