# Eidra Launch Checklist

Use this checklist to prepare a soft launch, creator outreach wave, or public release.

## 1. Product Readiness

- README is current and accurate
- quickstart works
- `eidra doctor` works on a clean machine
- `eidra setup` outputs useful guidance
- demo GIF reflects the current product
- screenshots reflect the current UI

## 2. Messaging Readiness

- one-line description is finalized
- short description is finalized
- beginner explanation is linked
- developer explanation is linked
- architecture page is linked
- media kit is usable without rewriting

## 3. Outreach Readiness

- DM templates are prepared
- first 10 warm contacts are listed
- creator and newsletter list exists
- each target has one clear reason for outreach
- first link for each target is chosen
- follow-up schedule is defined

## 4. Social Readiness

- short launch post is drafted
- longer launch post is drafted
- thread outline is drafted
- one static screenshot is ready
- one short demo clip or GIF is ready

## 5. GitHub Readiness

- README has clear audience paths
- docs are linked from the main README
- multilingual README navigation is current
- AGENTS.md is current
- contributing and security docs are easy to find
- About text is finalized
- topics are set
- social preview image is ready
- first release notes are drafted
- discussion announcement is drafted

## 6. Launch Day Sequence

1. Publish or pin the main repo update
2. Send warm DMs first
3. Send targeted creator and newsletter outreach
4. Post the public announcement
5. Reply quickly to early questions and confusion

## 7. Post-Launch Loop

- collect the first 10 reactions
- tag feedback as setup, positioning, feature, or docs
- improve the fastest sources of confusion first
- update README and docs based on repeated questions
- reuse strong user quotes when permission is granted

## 8. Success Signals

- people understand the product quickly
- setup does not stall in the first 10 minutes
- replies include real feedback, not just compliments
- creators can describe Eidra in one sentence accurately
- developers know which doc to read next

## 9. Common Failure Modes

- launch copy is stronger than the actual product
- public posts go out before warm feedback arrives
- too many links are sent in first contact
- README and docs say slightly different things
- demo assets are out of date
